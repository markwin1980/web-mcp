# Git 提交信息规范

## 格式

```
<类型>: <范围> - <简短描述>

<详细描述（可选）>

<关联问题（可选）>
```

## 类型标识

| 类型 | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `refactor` | 代码重构（不改变功能） |
| `perf` | 性能优化 |
| `docs` | 文档更新 |
| `test` | 测试相关 |
| `chore` | 构建/工具/依赖更新 |
| `style` | 代码格式（不影响逻辑） |

## 示例

### 功能添加
```
feat: url_fetcher - 添加内容缓存机制

- 使用内存缓存减少重复请求
- 缓存 TTL 默认为 5 分钟
- 支持通过配置自定义缓存时间

Closes #12
```

### Bug 修复
```
fix: browser_service - 修复浏览器实例泄漏问题

确保页面关闭后正确释放资源
```

### 代码重构
```
refactor: mcp_stdio - 统一错误处理逻辑

将各模块的异常处理统一到 MCP 层
```

### 文档更新
```
docs: README - 更新安装说明
```

### 测试
```
test: web_search - 添加搜索结果解析测试
```

## 注意事项

- 简短描述不超过 50 字符
- 范围使用受影响的文件名或文件夹名
- 详细描述使用列表形式说明具体变更
- 关联问题使用 `Closes #issue_num` 格式
